⚡ MCP Bridge — Scoped Credential Handoff

Point AI tools at the guarded local bridge so they can request one approved credential by service name. macOS uses 127.0.0.1:3579; Open WebUI Docker uses host.docker.internal:3579. Do not dump the full vault into Hermes, chat prompts, Git, or browser session state.

# Add to Claude Code settings (~/.claude/settings.json): { "mcpServers": { "api-vault": { "command": "node", "args": ["/Users/Ace/Codex/apps/api-vault-mcp/index.js"], "env": { "VAULT_PATH": "/Users/Ace/.api-vault/keys.enc" } } } } # Health checks: # macOS / Claude Code: curl http://127.0.0.1:3579/health # Open WebUI Docker: http://host.docker.internal:3579 # # Then in any Claude/Hermes session: # "request SENDY_API_KEY for newsletter send test" → operator-approved single-key handoff # "request GOOGLE_PLACES_KEY for Shop Local import" → one service, one task # Bulk "all keys" export is intentionally blocked for agent use.
